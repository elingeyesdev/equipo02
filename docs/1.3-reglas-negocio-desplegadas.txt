ENTREGABLE 1.3: REGLAS DE NEGOCIO DESPLEGADAS

Objetivo del entregable
Consolidar la implementación de la red Hyperledger Fabric con el despliegue de los contratos inteligentes (Chaincode) que gestionan los activos (Clientes) y las operaciones de valor (Tokens), validando su correcto funcionamiento desde la terminal de comandos (CLI) y la interfaz de datos (CouchDB).

1. RESUMEN TÉCNICO
Configuración del entorno final:
- Red: Test-Network (v2.5.x)
- Canal: clientes
- Nodos: peer0.org1 (localhost:7051), peer0.org2 (localhost:9051)
- Consenso: Raft (Orderer en puerto 7050)
- Endorsement Policy: Mayoría de miembros (Org1 Y Org2 deben firmar)
- Lenguaje Chaincode: Go (v1.22.5)

2. REGLAS DE NEGOCIO - ASSET (CLIENTE)
Se utiliza el contrato "cliente_cc" (basado en asset-transfer-basic) con el siguiente mapeo lógico:

| Función en Chaincode | Lógica de Negocio   | Descripción                                          |
| :---                 | :---                | :---                                                 |
| CreateAsset          | Alta de Cliente     | Crea el registro único del cliente en el ledger.     |
| ReadAsset            | Consulta de Cliente | Busca y devuelve el JSON del cliente por su ID.      |
| GetAllAssets         | Listado de Clientes | Recupera todos los clientes registrados en el canal. |

Validaciones aplicadas:
- Unicidad: No se puede registrar un clienteId que ya exista.
- Consenso: Se requiere la firma de ambos peers para completar la escritura.

3. REGLAS DE NEGOCIO - TOKEN (VALOR)
Se utiliza el contrato "token_erc20" (basado en token-erc-20) con el siguiente mapeo lógico:

| Función en Chaincode | Lógica de Negocio   | Descripción                                          |
| :---                 | :---                | :---                                                 |
| Initialize           | Activación          | Define el nombre (TokenProyecto) y símbolo del token (TOK). |
| Mint                 | Emisión             | Crea unidades de valor digitales para un cliente.    |
| Transfer             | Transferencia       | Envía valor entre dos cuentas validando el saldo disponible. |
| BalanceOf            | Consultar Saldo     | Indica el monto acumulado de un cliente específico.    |

4. COMANDOS DE DESPLIEGUE (EJECUTADOS)

Despliegue de Asset:
./network.sh deployCC -ccn cliente_cc -ccp ../asset-transfer-basic/chaincode-go/ -ccl go -c clientes

Despliegue de Token:
./network.sh deployCC -ccn token_erc20 -ccp ../token-erc-20/chaincode-go/ -ccl go -c clientes

Invocación de Activación (Token):
peer chaincode invoke ... -n token_erc20 -c '{"function":"Initialize","Args":["TokenProyecto","TOK","2"]}'

6. GUÍA PARA EL EQUIPO (PRUEBAS ADICIONALES)

Si algún compañero desea registrar un nuevo cliente para sus pruebas, debe seguir esta plantilla. 

IMPORTANTE: El comando exige "Doble Firma". Si olvidas uno de los dos '--peerAddresses', la Blockchain rechazará el registro por falta de consenso.

Template de Creación (Copiar y pegar cambiando los datos):
---------------------------------------------------------
peer chaincode invoke -o localhost:7050 --ordererTLSHostnameOverride orderer.example.com --tls --cafile "${PWD}/organizations/ordererOrganizations/example.com/orderers/orderer.example.com/msp/tlscacerts/tlsca.example.com-cert.pem" -C clientes -n cliente_cc \
  --peerAddresses localhost:7051 --tlsRootCertFiles "${PWD}/organizations/peerOrganizations/org1.example.com/peers/peer0.org1.example.com/tls/ca.crt" \
  --peerAddresses localhost:9051 --tlsRootCertFiles "${PWD}/organizations/peerOrganizations/org2.example.com/peers/peer0.org2.example.com/tls/ca.crt" \
  -c '{"function":"CreateAsset","Args":["ID_UNICO","COLOR","TALLA","NOMBRE_DUEÑO","VALOR"]}'

Template de Consulta (Para verificar):
--------------------------------------
peer chaincode query -C clientes -n cliente_cc -c '{"function":"ReadAsset","Args":["ID_UNICO"]}'
